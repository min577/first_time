# 처음학개론 — DCA 출품용 웹 프로토타입 개발 스펙

> 제43회 DCA 공모전 · 처음처럼 브리프 · Digital Campaign 카테고리
> 심사위원이 브리프보드의 QR/링크로 접속해 60초 안에 캠페인 구조를 체험하는 데모.
> 이 문서를 프로젝트 루트에 두고 Claude Code가 참조하도록 한다.

---

## 0. 한 줄 정의

**"병뚜껑을 따면 열리는 수업"** — 상황별 '선배의 한 줄' 가이드를 읽고, 익명으로 나의 처음을 고백하면, 사람들이 그 처음에 잔을 들어주는 UGC 플랫폼의 인터랙티브 프로토타입.

## 0.5 진입 채널 전략 (캠페인 설정 — 프로토가 재현해야 하는 세계관)

| 물량 | 진입 방식 | 비고 |
|---|---|---|
| 전 물량 | 뚜껑 안 QR | 비용 제로, 개봉해야 보임 = 구매·음용 순간과 접속 결합 |
| 한정판·팝업·굿즈 | **터치캡** — 전도성 접점이 있는 뚜껑을 화면에 도장처럼 찍으면 입장 | '출석 도장을 뚜껑으로 찍는 수업' 세계관. 도장 N회 적립 = 개근상 굿즈 |

프로토타입 스플래시는 **터치캡을 시뮬레이션**한다(아래 5번 Splash 명세). 데모에서 심사위원이 손가락으로 직접 '찍어보는' 경험이 기억 포인트.

## 1. 목표와 비목표

### 목표
- 심사위원이 설명 없이 30초 안에 핵심 루프(열람 → 고백 → 잔 들기 → 라벨 인쇄)를 이해한다
- 모바일(QR 스캔)과 데스크톱(URL 직접 입력) 모두에서 완성도 있게 보인다
- 시연 안정성: 새로고침해도 깨지지 않고, 네트워크 없이도 시드 데이터로 동작한다

### 비목표 (하지 않는 것)
- 실제 백엔드/DB/로그인 — localStorage로 충분
- 실제 결제, 실제 커뮤니티 운영
- SEO 최적화 (OG 태그만 챙긴다 — 심사위원 간 링크 공유 대비)

## 2. 기술 스택

| 항목 | 선택 | 이유 |
|---|---|---|
| 프레임워크 | React 18 + Vite | SPA 프로토타입에 최적, Vercel 배포 zero-config |
| 언어 | TypeScript | Claude Code 작업 시 타입이 곧 명세 역할 |
| 라우팅 | react-router-dom v6 | 화면별 URL 필요 (심사위원에게 특정 화면 딥링크 가능) |
| 스타일 | vanilla CSS + CSS 변수 (`src/styles/tokens.css`) | 디자인 토큰 직접 통제, Tailwind 미사용 |
| 애니메이션 | framer-motion | 뚜껑 팝, 잔 들기 +1, 화면 전환 |
| 상태 | React useState/useReducer + localStorage | 전역 스토어 불필요 |
| 배포 | Vercel (GitHub 연동, main 브랜치 자동 배포) | |

```bash
npm create vite@latest cheoeum-hakgaeron -- --template react-ts
cd cheoeum-hakgaeron
npm i react-router-dom framer-motion
```

## 4. 라우팅과 화면 구조

```
/            스플래시 (병뚜껑 입장)
/app         메인 셸 (하단 탭: 수강신청 / 고백실 / 건배사)
/app/courses           수강신청 — 카테고리·강의 목록
/app/courses/:slug     강의실 — 선배의 한 줄 + 한 줄 남기기
/app/confess           고백실 — 피드 + 작성
/app/toast             건배사 생성기
/about       캠페인 소개 (심사위원용 — 아이디어 요약 1페이지)
```

### 반응형 전략 (중요)
- **모바일 (≤480px)**: 앱이 화면 전체를 사용. QR 스캔 유입 기준 경험.
- **데스크톱 (>480px)**: 중앙에 375px 폰 프레임(라운드 28px, 그림자)으로 앱을 렌더하고,
  **왼쪽에 심사위원용 사이드 패널**을 배치한다:
  - 캠페인명 · 한 줄 컨셉 · 참여 루프 4단계 요약 · "브리프보드로 돌아가기" 없이 /about 링크
  - 이 패널이 있으면 데스크톱으로 접속한 심사위원이 컨텍스트를 잃지 않는다
- 구현: `<DeviceFrame>` 레이아웃 컴포넌트가 뷰포트 폭에 따라 분기.

### 핵심 인터랙션 명세

**터치캡 입장 (Splash) — 시그니처 1**
- 초록 풀스크린. 상단 "개교 20주년 · 처음처럼", 로고타입 "처음학개론"(Gowun Batang 40px), 부제 "교수는 없고, 선배만 있는 수업"
- 중앙에 출석부 느낌의 빈 도장 자리(원형 점선, 130px) + 카피 "**뚜껑을 화면에 대보세요**"
- **터치캡 감지**: `touchstart`에서 `e.touches.length >= 3` 이고 세 좌표가 반경 ≈300px 원 안에 들어오면 인식 성공
  → 도장 스탬프 애니메이션: 뚜껑 자국(원형 + "처음처럼" 각인)이 scale 1.4→1로 쾅 찍히고(0.25s, ease-out) 잉크 번짐(초록 링 확산 0.5s) + 짧은 햅틱(`navigator.vibrate(30)`, 지원 시)
  → "출석 완료" 표시 0.6s 후 navigate('/app/courses')
- **폴백 (필수)**: 단일 탭/클릭도 동일하게 동작. 세 손가락은 요구 조건이 아니라 이스터에그 — 접근성상 어떤 입력으로도 입장 가능해야 함
- 도장 자리 아래 캡션(12px): "한정판 터치캡을 화면에 찍어 입장하세요 · 일반 병은 뚜껑 안 QR로"
- `prefers-reduced-motion`: 스탬프/번짐 생략하고 즉시 전환
- 하단 고정: "만 19세 이상 · 지나친 음주는 처음도 망칩니다" (11px, opacity .55)

**잔 들기 (RaiseButton) — 시그니처 2**
- 기본: 빈 소주잔 SVG + "N잔" (테두리 --line)
- 탭: 잔이 앰버로 채워지며 `clink` 흔들림(rotate -14 → 9 → 0, 0.5s), "+1잔" 텍스트가 위로 떠오르며 사라짐(floatUp 0.7s), 카운트 +1
- 탭 후: --amber-pale 배경 + --amber 테두리로 고정, 재탭 불가 (localStorage에 raised id 저장)
- 숫자는 toLocaleString() 포맷

**고백실 (Confess)**
- 상단 히어로 카드(초록): "어른의 처음은 아무도 축하해주지 않는다" / "여기선 실토해도 됩니다. 익명이니까."
- Composer placeholder: "○○이 처음입니다…" / 보조문구 "익명 보장 · 삭제는 언제든"
- 작성 시 피드 최상단 prepend, 작성자 "익명의 나", 연초록 배경으로 내 글 구분, 내 글엔 삭제 버튼
- cheers ≥ 3000 카드에 LabelBadge("이번 주 라벨 인쇄 후보") — 참여→제품 회귀를 암시하는 장치

**강의실 (CourseRoom)**
- 한 줄 남기기 제출 시 작성자명 "나 (오늘부터 선배)" — 수혜자→기여자 전환을 카피로 못 박는다

**건배사 (ToastMaker)**
- 자리 선택(회식/동창회/축하자리) → "건배사 뽑기"(앰버 버튼) → 카드에 선창 문구 + "선창·후창 — 오늘이! 처음처럼!" 랜덤
- 다시 뽑기 가능. 카드 하단: "따라 읽기만 하면 됩니다. 나머지는 잔이 알아서."

**/about (심사위원용)**
- 1스크린 요약: Background → Idea(루프 4단계 다이어그램은 CSS로 간단히) → Effect
- 하단에 "프로토타입 체험하기 →" CTA로 / 재진입

## 6. 데이터와 영속화

- 시드는 `data/*.ts`에 하드코딩. 사용자 생성분은 localStorage 키:
  - `chg.confessions` (배열), `chg.tips.{slug}` (배열), `chg.raised` (id Set)
- `useLocalList(key, seed)` 훅: 시드 + 로컬분 병합, add/remove 제공, JSON 직렬화 예외 try-catch
- **리셋 이스터에그**: /about 하단에 작은 "데모 초기화" 버튼 (시연 전 초기화용)

## 7. 품질 기준

- 접근성: 모든 버튼 aria-label, 폰트 최소 11px, 초록 배경 위 텍스트 대비 4.5:1 이상
- 모션: `prefers-reduced-motion` 시 뚜껑/잔 애니메이션 즉시 완료 처리
- 성능: 이미지 없이 SVG·CSS로만 (LCP < 1.5s 목표), 폰트 display=swap
- OG 태그: title "처음학개론 — 교수는 없고, 선배만 있는 수업", description 1줄, og:image는 `/public/og.png` (1200×630, 초록 배경 + 로고타입 — 마지막에 제작)
- 파비콘: 병뚜껑 모티프 SVG

## 8. 배포 (Vercel)

1. GitHub 리포 생성 → push
2. vercel.com → Import Project → 프레임워크 자동 감지(Vite) → Deploy
3. SPA 라우팅 404 방지: `vercel.json`
   ```json
   { "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }] }
   ```
4. 프로젝트명을 `cheoeum-hakgaeron`으로 → `cheoeum-hakgaeron.vercel.app`
5. 브리프보드에는 이 URL + QR(무료 QR 생성기, 오류복원 H) 삽입
6. **출품 전 체크**: 시크릿 창 + 폰 실기기에서 QR 스캔 테스트, **iOS Safari와 Android Chrome 각각에서 세 손가락 터치캡 입장 테스트**(브라우저 제스처와 충돌 시 `touch-action: none`을 도장 존에 적용), 마감(7/30) 전에 배포 잠금(main 브랜치 더 이상 push 금지)

## 9. 작업 순서 (Claude Code 세션 플랜)

1. 스캐폴딩 + 토큰 + 라우팅 뼈대 (빈 페이지들)
2. DeviceFrame + Splash(뚜껑 애니메이션)
3. 수강신청 → 강의실 (시드 데이터, TipCard, RaiseButton)
4. 고백실 (Composer, localStorage 훅)
5. 건배사 생성기
6. /about + OG + 파비콘 + reduced-motion + 폴리싱
7. Vercel 배포 + 실기기 QA

각 단계 끝날 때마다 `npm run dev`로 눈 확인 후 커밋. 커밋 메시지는 한국어로 간단히.

## 부록 D. /about 페이지 카피 (브리프보드와 동일 논리)

- **Background**: 회식 문화 축소로 소주의 사회적 역할이 줄었지만, 인생의 '처음'은 사라지지 않았다. 2030의 처음은 몰라서 무섭고, 3049의 처음은 모른다고 말할 수 없다.
- **Idea**: 병뚜껑을 따면 열리는 수업, 처음학개론. 뚜껑 안 QR — 한정판은 뚜껑을 화면에 도장처럼 찍어 — 입장해, 선배의 한 줄을 읽고 → 나의 처음을 고백하면 → 사람들이 잔을 들어주고 → 베스트 고백은 병 라벨에 인쇄된다.
- **Effect**: 오늘의 후배가 내일의 선배가 되는 자생 루프. 하나의 언어('처음')로 3049와 2030을 관통하며, 절주 가이드 내장으로 책임 있는 음주 문화까지 설계한다.
