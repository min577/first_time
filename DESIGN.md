# 처음학개론 디자인 시스템

## 방향

처음학개론은 대학 교재를 흉내 낸 전시물이 아니라, 처음 겪는 술자리에서 바로 꺼내 보는 모바일 도구다. 병초록과 종이색은 브랜드 자산으로 유지하고, 기능 UI는 짧고 분명하게 쓴다.

- 장르: 절제된 playful consumer app
- 주조색: 병초록
- 표면색: 따뜻한 종이색
- 타이포: 기능 UI와 본문은 Pretendard로 통일한다. 로고 이미지만 별도의 브랜드 서체 역할을 한다.
- 레이아웃: 모바일 단일 열을 기본으로 한다. 강의는 인덱스, 강의 상세는 긴 문서, 고백은 일반 세로 목록 구조를 쓴다.
- 모션: 버튼 눌림과 결과 교체에만 사용한다. 스크롤 스냅과 반복 장식 모션은 사용하지 않는다.

## 카피 원칙

1. 한 문장에는 한 가지 정보만 담는다.
2. 사용자가 실제로 할 수 있는 행동만 안내한다.
3. `안내서`, `과목`, `n줄`처럼 콘셉트를 설명하기 위한 메타 표현보다 `강의`, `조언`, `잔 들기`처럼 행동을 바로 이해할 수 있는 말을 쓴다.
4. 위로를 강요하거나 감정을 단정하지 않는다.
5. 버튼은 결과가 드러나는 동사형으로 쓴다.

## 토큰 계약

- 모든 CSS 색상은 `src/styles/tokens.css`의 이름 있는 토큰을 사용한다.
- 기능 텍스트는 `--font-ui`를 사용한다.
- 최소 본문 크기는 16px, 보조 문구는 14px로 한다.
- 터치 가능한 버튼과 링크는 최소 44px 높이를 확보한다.
- 모서리는 `--radius-control`, `--radius-card`, `--radius-sheet` 세 단계만 사용한다.
- 포커스 링은 모든 키보드 인터랙션에 즉시 표시한다.

## 화면 구조

### 시작

브랜드 문장, 로고, 시작 버튼 순서다. 실제 하드웨어로 인식되지 않는 터치캡 제스처를 요구하지 않는다.

### 강의 목록

상단 소개와 대상 필터 다음에 해당 강의만 노출한다. 카드에는 강의 코드, 제목, 이동 화살표만 둔다.

### 강의 상세

강의 요약, 선배들의 조언, 조언 남기기 순서로 읽힌다. 고백 이동은 보조 행동이다.

### 처음 고백

벽화 소개 다음에 고백 카드가 자연스럽게 이어지는 일반 세로 목록이다. 정렬, 잔 들기, 조언, 포토카드, 고백 남기기의 터치 영역을 44px 이상으로 유지한다.

### 건배사

자리 선택, 건배사 보기, 결과 복사 순서다. 결과가 나온 뒤에는 복사가 주 행동이고 다시 뽑기는 보조 행동이다.

## 반응형 기준

320px, 375px, 414px, 768px에서 가로 스크롤이 없어야 한다. 버튼 문구는 두 줄로 갈라지지 않아야 하며, 데스크톱에서는 심사용 패널과 375px 데모 프레임을 유지한다.
